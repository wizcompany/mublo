# 문제 해결

설치와 운영 중 자주 만나는 문제와 해결 방법을 정리했습니다.

## 설치 관련

### 설치 페이지가 안 보인다

- 웹 루트가 프로젝트 루트가 아니라 `public/` 을 바라보는지 먼저 확인합니다.
- 설치 전에는 `config/` 와 `storage/` 가 없을 수 있지만, `public/index.php` 와 `public/install/index.php` 는 웹서버에서 접근 가능해야 합니다.
- Apache/Nginx 설정에서 PHP 처리 대상이 `public/` 아래로 잡혀 있는지 확인합니다.

### DB 연결이 실패한다

- 설치 화면의 DB 호스트, 포트, DB명, 계정, 비밀번호를 다시 확인합니다.
- 기본값은 보통 `localhost` 이지만, Docker나 외부 DB를 쓰면 실제 호스트명을 넣어야 합니다.
- 대상 DB 계정에 테이블 생성 권한이 있는지 확인합니다.
- MySQL 5.7+, MariaDB 10.3+ 환경인지 점검합니다.

### `storage/` 퍼미션 오류

- 설치 후 생성되는 `storage/` 디렉토리에 웹서버 쓰기 권한이 있어야 합니다.
- 파일 캐시, 세션, 로그가 모두 `storage/` 를 사용하므로 권한이 부족하면 관리자 로그인, 업로드, 캐시가 함께 실패할 수 있습니다.
- Linux 환경이라면 웹서버 계정(`www-data`, `apache`, `nginx` 등)에 쓰기 권한이 있는지 확인합니다.

## 운영 관련

### 관리자 로그인이 안 된다

- 설치 직후 만든 관리자 계정으로 로그인하는지 확인합니다.
- 세션이 유지되지 않으면 `storage/sessions/` 권한과 `SESSION_DRIVER` 설정을 먼저 점검합니다.
- 여러 도메인 또는 프록시 환경에서는 실제 접속 도메인과 앱 설정이 일치하는지 확인합니다.

### 페이지가 깨져 보인다

- 브라우저 개발자 도구에서 CSS/JS가 404로 내려오는지 확인합니다.
- 웹 루트가 `public/` 이 아니면 정적 자산 경로가 모두 깨질 수 있습니다.
- 캐시 문제일 수 있으므로 브라우저 강력 새로고침 후 다시 확인합니다.

### 이미지 업로드가 안 된다

- 업로드 디렉토리 쓰기 권한을 확인합니다.
- PHP `upload_max_filesize`, `post_max_size` 제한이 너무 작지 않은지 확인합니다.
- 공개 업로드 경로가 웹서버에서 정상 노출되는지 확인합니다.

### 메일이 발송되지 않는다

- `.env` 의 `MAIL_DRIVER`, `MAIL_FROM_ADDRESS`, SMTP 항목을 다시 확인합니다.
- SMTP 사용 시 `MAIL_SMTP_HOST`, `MAIL_SMTP_PORT`, `MAIL_SMTP_USERNAME`, `MAIL_SMTP_PASSWORD`, `MAIL_SMTP_ENCRYPTION` 이 모두 맞아야 합니다.
- 먼저 PHP mail 드라이버 대신 SMTP로 명시 설정해서 테스트하는 편이 문제 파악이 쉽습니다.

## 도메인 관련

### 도메인 추가 후 접속이 안 된다

- 웹서버 가상호스트 또는 리버스 프록시 설정에 해당 도메인이 실제로 연결되어 있는지 확인합니다.
- DNS 전파가 끝나지 않았거나, HTTPS 인증서 설정이 빠졌을 수 있습니다.
- 관리자에서 도메인을 등록해도 웹서버가 그 도메인을 받지 않으면 접속할 수 없습니다.

### 도메인별 설정이 적용되지 않는다

- 현재 요청이 기대한 도메인으로 들어오는지 확인합니다.
- 관리자에서 수정한 대상 도메인이 맞는지 다시 확인합니다.
- 캐시 사용 중이면 설정 반영 후 캐시를 비우고 다시 확인합니다.

## 성능 관련

### 페이지 로딩이 느리다

- 개발 환경에서 `APP_DEBUG=true` 인지 확인합니다. 운영에서는 `false` 를 권장합니다.
- 파일 캐시를 기본으로 쓰고 있는지, Redis를 쓴다면 연결 오류 없이 정상 동작하는지 점검합니다.
- 통계, 배너, 위젯, 쇼핑몰 목록처럼 데이터가 많은 화면은 초기 데이터 양에 따라 느려질 수 있으므로 쿼리와 인덱스를 함께 확인합니다.

### 캐시 설정 (`file` / `redis`)

- 기본값은 `file` 이며 별도 준비 없이 시작할 수 있습니다.
- Redis를 쓰려면 `.env` 에 `CACHE_DRIVER=redis`, `SESSION_DRIVER=redis` 와 `REDIS_HOST`, `REDIS_PORT`, `REDIS_PASSWORD` 를 맞춰야 합니다.
- Redis 연결 실패 시 세션 손실이나 캐시 미작동이 함께 발생할 수 있습니다.

## 로그 확인

### 에러 로그 위치

- 애플리케이션 로그는 기본적으로 `storage/logs/` 아래를 먼저 확인합니다.
- 웹서버 에러는 Apache/Nginx/PHP-FPM 로그도 함께 봐야 원인을 빨리 찾을 수 있습니다.

### 로그 레벨 설정

- 개발 중에는 상세 로그를 남기고, 운영에서는 민감 정보가 남지 않도록 로그 노출 범위를 조절합니다.
- 문제 재현 시에는 웹서버 로그와 애플리케이션 로그를 같은 시각 기준으로 함께 비교하는 편이 좋습니다.
