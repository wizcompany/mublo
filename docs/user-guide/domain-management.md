# 멀티 도메인 관리

## 멀티 도메인이란?

하나의 Mublo 설치로 여러 개의 독립적인 사이트를 운영할 수 있습니다. 각 도메인은 회원, 설정, 테마, 권한, 콘텐츠가 완전히 분리됩니다.

```
하나의 서버
├── example.com       → 사이트 A (회원/설정/테마 독립)
├── shop.example.com  → 사이트 B (회원/설정/테마 독립)
└── blog.example.com  → 사이트 C (회원/설정/테마 독립)
```

## 도메인 추가

관리자 > 도메인 관리 > 도메인 추가

### 기본 정보

| 항목 | 설명 |
|------|------|
| 도메인명 | `shop.example.com` (DNS에 등록된 도메인) |
| 사이트 제목 | 브라우저 탭에 표시 |
| 상태 | 활성/비활성/차단 |
| 계약 유형 | 무료/월간/연간/영구 |
| 계약 기간 | 시작일 ~ 종료일 |

### 제한 설정

| 항목 | 설명 | 기본값 |
|------|------|--------|
| 저장소 제한 | 업로드 파일 용량 한도 | 1GB |
| 회원 수 제한 | 등록 가능 회원 수 | 무제한 |

### DNS 설정

도메인을 추가하기 전에 DNS에서 해당 도메인이 서버 IP를 가리키도록 설정해야 합니다.

```
shop.example.com  → A 레코드 → 서버 IP
```

## 도메인별 설정

각 도메인은 독립적인 설정을 가집니다.

### 사이트 설정 (siteConfig)
- 사이트 제목, 부제목
- 로고 이미지 (PC/모바일)
- 파비콘, OG 이미지

### 회사 정보 (companyConfig)
- 회사명, 대표자, 사업자등록번호
- 주소, 전화번호

### SEO 설정 (seoConfig)
- 메타 설명, 키워드
- 검색엔진 노출 여부

### 테마 설정 (themeConfig)
- 프론트 프레임 스킨
- 관리자 스킨

### 확장 설정 (extensionConfig)
- 활성화된 패키지 목록
- 활성화된 플러그인 목록
- 도메인별로 다른 패키지/플러그인 조합 가능

## 도메인 격리

| 대상 | 격리 방식 |
|------|----------|
| 회원 | `domain_id`로 완전 분리. A 도메인 회원이 B에 로그인 불가 |
| 게시판 | 도메인별 독립 게시판, 게시글 |
| 블록/페이지 | 도메인별 독립 메인 페이지 |
| 메뉴 | 도메인별 독립 네비게이션 |
| 파일 | `public/storage/D{domainId}/` 하위에 분리 저장 |
| 캐시 | 도메인 ID 기반 캐시 키 분리 |
| 세션 | 도메인별 세션 이름 분리 |

## 도메인 그룹

도메인 간 계층 구조를 설정할 수 있습니다 (`domainGroup` 필드). 상위 도메인 관리자가 하위 도메인을 관리하는 용도로 사용합니다.

## 도메인 상태

| 상태 | 설명 |
|------|------|
| active | 정상 운영 |
| inactive | 비활성 (관리자만 접근 가능) |
| blocked | 차단 (모든 접근 불가) |

계약 기간이 만료되면 자동으로 접근이 제한됩니다.
