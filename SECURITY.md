# Security Policy

보안 취약점은 공개 이슈로 올리지 않는 것을 권장합니다.

## 신고 방법

- 재현 방법
- 영향 범위
- 예상 악용 시나리오
- 임시 완화책이 있으면 함께 정리해서 전달해 주세요.

현재는 저장소 관리자에게 비공개 채널로 먼저 전달하는 방식을 권장합니다.

## 공개 이슈로 올려도 되는 항목

- 일반 버그 리포트
- 문서 오류
- 설치 가이드 보완
- 보안과 직접 관련 없는 설정 문제

## 공개 이슈로 올리면 안 되는 항목

- 실제 토큰, 키, 비밀번호가 포함된 내용
- 재현 가능한 공격 절차
- 운영 환경 URL, 관리자 계정, 내부 인프라 정보

## 처리 원칙

- 취약점 확인 후 재현 가능 여부를 먼저 검토합니다.
- 수정이 준비되기 전에는 세부 악용 정보 공개를 미룹니다.
- 수정 후에는 필요한 범위에서 원인과 대응 내용을 문서화합니다.
