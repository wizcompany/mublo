# Mublo Framework

**멀티 도메인 사이트와 운영 기능을 반복해서 다시 만들지 않기 위한 PHP 플랫폼**

사이트를 여러 개 운영하다 보면 게시판, 회원, 팝업, 포인트, 통계, 설문, 소셜 로그인, 쇼핑몰 같은 기능을 프로젝트마다 다시 만들게 됩니다.

Mublo Framework는 이런 반복을 줄이기 위해 만든 멀티 도메인 PHP 플랫폼입니다. 하나의 코어 위에 패키지와 플러그인을 조합해 사이트를 구성하고, 운영 기능은 관리자에서 다루고, 필요한 비즈니스 기능은 확장 모듈로 붙일 수 있도록 설계했습니다.

## 왜 필요한가

- 여러 사이트를 운영할수록 코드베이스가 갈라지고 공통 기능이 중복 구현되기 쉽습니다.
- 일반적인 CMS는 빠르게 시작하기 좋지만, 기능 확장과 구조 제어가 답답할 수 있습니다.
- 일반적인 프레임워크는 자유롭지만, 운영 기능을 처음부터 다 쌓아야 하는 부담이 큽니다.
- 실무에서는 게시판, 회원, 팝업, 통계, 포인트, 쇼핑몰 같은 기능이 한 프로젝트 안에서 함께 필요해지는 경우가 많습니다.

## 무엇을 해결하는가

- **멀티 도메인 운영**: 하나의 코드베이스로 여러 사이트를 운영하고, 도메인별 설정/회원/권한을 분리합니다.
- **운영 기능의 공통화**: 관리자, 페이지 구성, 게시판, 팝업, 배너, 통계 같은 기능을 공통 기반으로 제공합니다.
- **확장 구조 제공**: Plugin / Package 구조로 기능을 분리해 필요한 모듈만 붙일 수 있습니다.
- **실무형 개발 흐름**: Controller → Service → Repository → Entity 구조, DI 컨테이너, 이벤트 시스템, Result 패턴을 사용합니다.
- **낮은 진입 장벽**: PHP만으로 동작하며 별도의 Node.js 빌드 과정 없이 시작할 수 있습니다.

## 이런 경우에 적합합니다

- 여러 사이트를 한 기반에서 운영하려는 팀
- 관리자 기능이 포함된 PHP 기반 플랫폼이 필요한 팀
- 단순 CMS보다 구조 제어가 필요하지만, 완전한 풀커스텀은 부담인 팀
- 게시판형 사이트, 비즈니스 사이트, 쇼핑몰을 같은 확장 모델 위에서 운영하려는 경우

## 대표 사용 시나리오

### 1. 회사/브랜드 사이트 운영

- 블록 기반 페이지 구성
- 배너, 팝업, 위젯, FAQ 조합
- 방문자 통계와 설문으로 운영 데이터 수집

### 2. 커뮤니티/게시판 중심 사이트

- 게시판 그룹/카테고리 운영
- 댓글, 첨부파일, 리액션, 포인트 연동
- 회원 기능과 소셜 로그인을 함께 사용

### 3. 운영 기능이 포함된 쇼핑몰

- 상품, 장바구니, 주문, 결제, 쿠폰, 배송 관리
- 회원 포인트와 리뷰/문의 기능 결합
- 기본 제공 PG 게이트웨이와 이벤트 확장 구조를 함께 사용

## 요구사항

- PHP 8.2 이상
- MySQL 5.7 이상 (또는 MariaDB 10.3 이상)
- PHP 확장: PDO, JSON, mbstring, openssl, gd

## 빠른 시작

```bash
# 1. 파일을 웹 서버 디렉토리에 배치
# 2. 브라우저에서 설치 페이지 접속
https://your-domain.com/install

# 3. 웹 설치기의 안내에 따라 DB 설정 → 도메인 설정 → 보안 설정 → 관리자 계정 생성 → 완료
```

설치가 완료되면 `/admin`으로 관리자 화면에 접속할 수 있습니다.

> 상세 설치 가이드: [docs/user-guide/installation.md](docs/user-guide/installation.md)

## 현재 공개 범위

이 저장소에는 아래 공개 모듈만 포함되어 있습니다.

- Core
- Package: `Board`, `Shop`
- Plugin: `Banner`, `Faq`, `MemberPoint`, `Popup`, `SnsLogin`, `Survey`, `VisitorStats`, `Widget`

아래 항목은 현재 공개 저장소에 포함하지 않습니다.

- 내부 운영용 모듈
- 리팩토링 전 `repack-*` 계열 패키지
- 사내/실험용 플러그인
- 선택 애드온 예시 (`TestPay` 등)

## 핵심 구조

```
┌─────────────────────────────────────────────┐
│                   Core                      │
│   라우팅, 인증, 세션, DB, 렌더링, 이벤트     │
│                                             │
│   ┌─────────────────────────────────────┐   │
│   │       멀티 도메인 (Context)          │   │
│   │     도메인별 설정, 테마, 회원 분리     │   │
│   └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
         ↑ 이벤트                  ↑ 이벤트
┌────────────────┐      ┌─────────────────────┐
│    Plugin      │      │      Package        │
│  (Core 기능    │      │  (독립 애플리케이션)  │
│   확장)        │      │                     │
└────────────────┘      └─────────────────────┘
```

## 디렉토리 구조

```
framework/
├── config/          # 빈 디렉토리 포함, 설정 파일은 설치 시 생성
├── database/        # Core 마이그레이션
├── docs/            # 문서
├── packages/        # 패키지 (Board, Shop 등)
├── plugins/         # 플러그인 (Banner, FAQ 등)
├── public/          # 웹 루트 (index.php, install/, storage/)
├── src/             # Core 소스
├── storage/         # 캐시, 로그, 세션, 임시 파일
├── tests/           # 테스트
└── views/           # 뷰 템플릿
```
